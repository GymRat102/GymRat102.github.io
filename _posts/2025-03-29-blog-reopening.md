又重新把自己的博客收拾搭建起来了，希望这一次能够坚持记录下去。

之前的博客之路也经历过一些折腾：最早用 Typlog，比较省心；后来想自己动手搭博客，切换到 Jekyll，用的是打包成 docker image 部署的方式；这之前又尝试过动态博客，作为一名 Java 程序员，选择了 Halo 这个项目，希望能够在过程中顺便探索一下这个开源项目。现在又回到了 Jekyll，并且更省心一点，直接使用 GitHub Pages 自动部署了。

我在决定用 Jekyll + GitHub Pages 来作为博客技术栈的时候，还是有一些考量的。

首先，静态博客确实省心很多，用 GitHub Pages 部署的话就更不用考虑服务器、部署和监控这一系列事情了。作为程序员确实是喜欢折腾，也有能力折腾这些。但是博客的初衷还是为了记录和写作，不想在工具本身上走的太远。

其次，我希望通过 git 提交留下博客更新的历史，以及通过 GitHub 让这份博客永久的保存下来（见 [People Die, but Long Live GitHub](https://laike9m.com/blog/people-die-but-long-live-github,122/)）。Blog the fleeting life，我希望多年之后回顾自己的博客的时候，从其更改和维护的历史上都能看到自己曾经的想法。

最后，静态博客千千万，Jekyll 给我的感觉是这个框架它原生提供的一些功能就非常适合作为博客站点，只是一个粗浅的印象，别的静态博客生成器例如 Hexo、Hugo 就没有给我留下这样的印象。之后研究深入之后可能会写一写这一点。

当然，使用静态博客写博客，还有一个重要的环节就是编辑器，需要有一个趁手（简洁、漂亮、高效）的工具和一个方便的工作流，否则 「创建文件-打开编辑器-（图片/视频的处理）-保存-预览-发布」 的这整个过程就会感觉像是 hassle，而不是像在网页端博客后台直接写那么流畅。目前我选择了 [Typora](https://typora.io/) 这款编辑器，用了 [Vercel](https://theme.typora.io/theme/Vercel/) 这款主题感觉中英文的书写都非常简洁好看。后续还需要解决工作流和图床的问题，再分享。
